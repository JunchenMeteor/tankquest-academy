# Phase 4 2.5D 训练场范围

> 状态：2026-07-16 开始实施，目标版本 v0.4.0。Phase 4 只升级 Web 训练场的资源管线和 2.5D 视觉表现，不重写既有战斗、学习或结算规则。

## 1. 技术决策

Phase 4 使用 Phaser 4.2.1 的分层绘制实现 2.5D 等距训练场，不接入 Three.js，也不切换到 Unity、Godot 或完整 3D 物理。

```text
后端关卡、坦克、涂装和资源清单
-> Web AssetClient 校验并预加载轻量描述资源
-> Phaser 在既有世界坐标上绘制等距地面、立体障碍和分层坦克
-> Arcade Physics 继续负责移动、射击、碰撞和命中
```

视觉层可以投影、加阴影和高度，但物理实体仍使用现有确定性坐标。鼠标瞄准、炮弹轨迹、撞击、穿深、敌人状态机和难度数值不得因表现升级而产生另一套规则。

选择这一方案的原因：

- 保留已经验证的 Phaser 战斗闭环和低端设备性能。
- 不向首屏再加入一套大型 3D 运行时。
- 让坦克轮廓、主题和涂装升级可以独立审查和降级。
- 后续若进入真正 low-poly 3D，仍可复用资源清单和客户端层。

## 2. 必须实现

### 2.1 资源清单

- 新增后端管理的资源元数据和公开只读 manifest API。
- manifest 至少包含 `assetId`、类型、版本、URL、SHA-256、字节数、标签、预览和依赖。
- 资源只关联已发布关卡；不存在、未发布或失效资源不能进入可用 manifest。
- Web 通过 `AssetClient` 获取、校验和预加载资源，不由 React 组件直接调用 `fetch`。
- manifest、网络或校验失败时使用内置轻量兜底表现，不阻断答题、战斗和结算。
- Phase 4 使用仓库内轻量 JSON/SVG 资源；对象存储和 CDN 迁移后置。

### 2.2 2.5D 场景

- 三类地图显示等距地面网格、可读边界、层次化装饰和立体障碍。
- 障碍的显示范围与 Arcade Physics 碰撞范围保持一致。
- 阴影、侧面和高度只属于表现层，不改变世界坐标或命中盒。
- `training-base`、`forest-camp`、`snow-field` 主题同时影响 React 外壳和 Phaser 场景。
- 切换主题后下一局使用新场景资源；当前局不热替换物理对象。

### 2.3 坦克识别度与涂装

- `star-shield`、`swift-fox`、`iron-mountain` 具有不同车体比例、炮塔尺寸、炮管长度和轮廓细节。
- scout、medium、heavy 敌人使用同一套角色轮廓语言，但保持训练敌方配色。
- 已装备涂装继续来自后端，主色、辅色、履带和高光在分层坦克上可辨识。
- 任务选择页提供与实际游戏轮廓一致的轻量坦克预览。
- 不使用真实军徽、敏感符号、商业模型复刻或写实战损。

### 2.4 性能与降级

- Phaser 继续按进入对局时懒加载。
- 轻量描述资源按关卡预加载，并限制资源数量和大小。
- Desktop 目标 60 FPS，低端笔记本和 Mobile Web 目标 30 FPS 以上。
- 不因表现资源失败产生未处理异常、黑屏或阻塞任务开始。
- 不增加服务器运行时服务；preview/release 继续使用现有四容器拓扑。

## 3. 权威边界

后端继续独立决定：

```text
正式关卡与题目
正确答案与错题解释边界
奖励、升级和持久化
坦克正式属性
敌人数量、能力和难度
结算与下一次练习建议
```

资源清单和 Phaser 只能决定如何显示这些数据，不能借由资源文件覆盖坦克属性、伤害、碰撞、题目、奖励或进度。

## 4. 明确不做

- 完整 3D 写实坦克、自由镜头、3D 物理或重做战斗系统。
- Three.js、React Three Fiber、Unity 或 Godot 接入。
- PWA、Tauri、Capacitor、应用商店发布或原生自动更新。
- 对象存储/CDN 迁移、大型 GLB、高清视频或付费美术包。
- 多人联机、商城、抽卡、付费数值或新的学习学科。
- 自动生成或自动发布儿童可见美术资源。

## 5. 验收标准

- manifest API 只返回允许的已发布资源，字段和 SHA-256 契约通过测试。
- `AssetClient` 对缺失、网络失败、无效 JSON、hash 不符和超限资源安全降级。
- 三种地图与三种 UI 主题具有可观察的 2.5D 场景差异。
- 三辆玩家坦克和三种敌人角色在缩小后的对局画面中仍可区分。
- 换涂装后选择页预览和下一局坦克同步变化。
- 现有移动、射击、碰撞、撞击伤害、穿深、敌方炮弹和结算回归测试继续通过。
- 无新增服务，preview/release 四容器健康，公开 Web/API/家长页面不回归。
- 格式、Prisma、lint、类型、单元、E2E、构建和三套容器检查通过。

## 6. PR 顺序

1. Phase 4 范围、技术决策和验收标准。
2. 资源元数据、manifest API、静态描述资源和 Web `AssetClient`。
3. 2.5D 等距训练场、立体障碍和三主题场景。
4. 高识别度玩家/敌方坦克、涂装预览和视觉回归测试。
5. Phase 4 验收证据、v0.4.0、preview/release 和生产验证。

每个 PR 必须从最新 `main` 创建，等待 `Verify` 和 API/Web/AI 三套容器检查通过后 squash merge；不得直接 push `main` 或 `release`。
