# 实施路线图

## 1. 开发原则

- 小 PR。
- 每个 PR 一个主题。
- 先闭环，再扩展。
- 不把 UI、后端、AI、资源管线混成一个巨大提交。

## 2. 阶段状态

| 阶段 | 状态 | 说明 |
| --- | --- | --- |
| Phase 1 工程 MVP | 已完成 | 三关卡选择、驾驶射击、3 道数学任务、后端结算、奖励升级和持久化闭环 |
| Phase 1 家庭试用 | 待验证 | 首次操作、3-5 分钟局长、重复游玩和家长理解度 |
| Phase 2+ | 未开始 | 后台、AI、资源管线、2.5D/3D 和多平台继续按独立 Issue/PR 推进 |

Phase 1 的逐项结论和非阻塞项见 `phase-one-mvp-acceptance.md`。家庭试用发现的问题进入独立 Issue，不回填到已冻结的工程 MVP 范围。

## 3. PR 拆分建议

### PR 1: 项目脚手架

- 前端 Vite + TypeScript。
- 后端基础服务。
- 数据库连接。
- 基础 CI。

### PR 2: 核心数据模型

- 用户/孩子。
- 坦克。
- 关卡。
- 题库。
- 对局。

### PR 3: 游戏 MVP

- Phaser 初始化。
- 坦克移动。
- 炮塔旋转。
- 射击。
- 碰撞。

### PR 4: 对局 API

- 创建对局。
- 提交答案。
- 提交事件。
- 结束结算。

### PR 5: 学习任务

- 数学题。
- 正确答案弹药箱。
- 错题解释模板。

### PR 6: 奖励和升级

- 星级。
- 零件。
- 坦克升级。

### PR 7: 后台最小版

- 题库管理。
- 关卡管理。
- 坦克管理。

### PR 8: 家长控制

- 孩子档案。
- 游玩时长。
- 难度范围。

### PR 9: AI 服务接入

- FastAPI AI 服务。
- 出题草稿。
- 错题解释。
- 安全校验。

### PR 10: 资源管线

- 资源 manifest。
- 模型/贴图加载。
- hash 校验。
- CDN/静态资源适配。

## 4. 每个 PR 必须说明

```text
改动范围
数据来源
新增 API
新增表
测试结果
风险
不做什么
```

## 5. 验证命令

具体命令按实际项目补充，但必须包含：

```text
lint
typecheck
unit test
contract test
build
e2e smoke
```
