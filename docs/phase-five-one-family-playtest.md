# Phase 5.1 家庭试用记录模板

> 状态：待执行。自动化门禁不能替代至少 3 组儿童与家长监督下的家庭试用；在三组匿名记录收口前，不得把 Phase 5.1 标记为“产品体验已验收”。

## 隐私边界

- 仅记录 `playtest-01`、`playtest-02`、`playtest-03` 等匿名编号。
- 不记录姓名、联系方式、学校、精确生日、照片、声音、地址或账号凭据。
- 年龄只使用宽泛阶段，例如 `6-8`、`9-11`；观察结论进入 Issue，不上传原始儿童信息。
- 家长全程监督并可随时停止；不把完成速度、错误或情绪作为儿童能力评价。

## 每组必填观察

| 项目 | 记录口径 |
| --- | --- |
| 首次操作障碍 | 是否能在无长说明下完成移动、瞄准、开火、答题和当前目标 |
| 单局时长 | 从开始任务到结算的约分钟数 |
| 重复游玩 | 是否主动选择重玩或下一任务，以及原因 |
| 升级感知 | 是否能说出或操作感受到至少一项升级前后差异 |
| 反馈理解 | 是否区分跳弹、未击穿、击穿、低血量和摧毁 |
| 家长理解度 | 是否能从报告理解练习内容、进步证据、关注项和下一步 |
| 设备与输入 | Web/PWA/Windows，键鼠或触控；不记录设备唯一标识 |
| 阻断问题 | 黑屏、卡死、无法继续、控件不可达、文本溢出或声音阻塞 |

## 收口标准

1. 三组记录均覆盖一局完整学习、战斗、结算、升级或重玩流程。
2. 阻断问题必须先建独立 Issue 并完成修复；非阻断观察按严重度进入后续范围。
3. 汇总只报告匿名趋势与可复现步骤，不引用可识别儿童身份的原话。
4. 自动化、preview/release 探针和三组家庭试用都完成后，才能关闭 Phase 5.1 父 Issue 并准备 v0.6.0。
