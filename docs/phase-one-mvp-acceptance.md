# 第一阶段 MVP 验收记录

验收日期：2026-07-14

## 1. 验收结论

第一阶段工程 MVP 通过验收，可以进入家庭试用。核心闭环已经形成并由后端权威判定：

```text
选择差异化关卡 -> 驾驶/射击 -> 完成 3 道数学任务并击败训练坦克
-> 后端结算星级和零件 -> 升级坦克
-> 刷新后在新对局加载升级效果
```

“孩子无说明可操作、一局 3-5 分钟、愿意连续玩 3 局、家长能理解学习内容”不能由代码或自动化测试代替，状态为家庭试用待验证，不阻塞工程 MVP 发布。

## 3. MVP 验收项

| 验收项 | 状态 | 验证方式 |
| --- | --- | --- |
| 无说明完成移动和射击 | 家庭试用待验证 | 观察儿童首次操作，不主动提示 |
| 一局 3-5 分钟 | 家庭试用待验证 | 记录至少 5 次首次游玩时长 |
| 至少 3 道题嵌入目标 | 通过 | E2E 连续完成 3 个补给挑战 |
| 通关后看到奖励 | 通过 | E2E 验证星级和 3 个炮管零件 |
| 奖励升级坦克属性 | 通过 | E2E 使用 2 个零件升级火力 |
| 升级后手感变化 | 通过 | 单元测试验证运行参数变化；刷新后 E2E 验证有效火力提升 |
| 刷新后进度不丢 | 通过 | PostgreSQL 持久化；E2E 刷新并启动新对局后读取升级属性 |
| 构建、测试、接口检查 | 通过 | CI 执行格式、schema、lint、typecheck、unit、build 和 Playwright |

家庭试用首轮反馈暴露的结算死路、属性不可感知、穿模、无撞击伤害、敌人单一和地图重复问题，已在 Phase 1.5 通过六个独立 Issue/PR 收口。对局现在必须同时完成学习任务和训练坦克目标后才能结算。

## 5. 家庭试用门槛

家庭试用至少记录 5 次儿童首次游玩和 3 次家长观察，回答以下问题：

- 是否能在不解释按键的情况下移动、瞄准和射击。
- 单局是否稳定在 3-5 分钟。
- 是否愿意连续完成至少 3 局。
- 是否能感知升级前后的弹速或装填差异。
- 家长是否能从结算页理解答题数量、正确数和奖励。

失败项进入独立 Issue，不在本阶段追加多人、AI、复杂后台、3D、移动端或商城功能。

## 6. 已知非阻塞项

- Vite 构建提示 Phaser 游戏 chunk 超过 500 kB，首屏拆包留到性能专项。
- Prisma 6 提示 `package.json#prisma` 配置将在 Prisma 7 移除，升级 Prisma 前迁移到独立配置文件。
- 当前家庭内测使用固定 demo 档案；正式鉴权、家长后台和多孩子管理属于后续阶段。
