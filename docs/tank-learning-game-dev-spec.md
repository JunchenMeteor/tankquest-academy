# 儿童坦克学习游戏开发文档

## 1. 项目定位

本项目是一款亲子/全年龄向的坦克闯关学习游戏。游戏以“驾驶经典风格坦克完成训练任务”为核心吸引点，通过战斗、探索、护送、打靶、机关解谜等玩法，让儿童、青少年和成人在游玩过程中完成数学、英语、方向、逻辑、知识问答、趣味测试等任务。

项目不是写实战争模拟，也不是简单答题软件。核心目标是把“坦克驾驶与养成的乐趣”和“可验证的学习/测试任务”结合起来。儿童模式优先强调安全、基础学习和正向反馈；成人模式可以提供更高难度知识题、策略题、反应挑战和轻量心理测试，但心理测试仅作为娱乐和自我观察，不提供医学、诊断或治疗结论。

### 目标用户

- 儿童用户：6-12 岁，重点是数学基础、英语词汇、方向感、逻辑推理、观察力、基础物理直觉
- 青少年用户：12-18 岁，重点是更复杂的逻辑、百科知识、策略规划、空间推理
- 成人用户：轻量休闲、知识挑战、亲子陪玩、趣味心理测试、反应和策略训练
- 陪伴用户：家长，可设置孩子题库、查看学习记录、参与亲子挑战
- 兴趣入口：喜欢坦克、车辆、炮塔、涂装、升级、收集、闯关和测试

### 核心体验

```text
选择坦克 -> 进入关卡 -> 驾驶/射击/完成学习任务
-> 获得星星/零件/徽章 -> 升级坦克/解锁涂装
-> 挑战更高难度关卡
```

## 2. 设计原则

1. 先好玩，再学习。
   学习任务必须嵌入战斗资源、开门、补给、奖励、路线选择中，不能做成强行弹窗答题。

2. 儿童友好。
   不使用血腥、真实战争叙事、阵营仇恨、真实战役包装。敌人可以是训练机器人、靶车、障碍无人车。

3. 低模但高识别度。
   坦克模型可以不是高清写实，但必须能看出经典坦克原型特征，包括车体轮廓、炮塔形状、炮管比例、履带比例、装甲倾斜和涂装。

4. 前后端完全分离。
   前端只展示 API 返回的数据和配置，不在组件里硬编码业务题库、坦克参数、奖励规则、关卡数据。

5. 配置驱动。
   题库、关卡、坦克参数、涂装、语言文案、主题皮肤均应配置化或落库，避免散落在 UI 组件中。

6. AI 是学习教练，不是游戏引擎。
   AI 用于出题、错题解释、难度调整、学习报告，不参与核心碰撞、移动、射击等实时游戏逻辑。

## 3. 核心玩法

### 3.1 关卡类型

| 类型 | 玩法 | 学习目标 |
| --- | --- | --- |
| 打靶训练 | 打中正确答案靶 | 数学、英语 |
| 护送补给 | 保护补给车到终点 | 路线规划、资源管理 |
| 清障任务 | 清除箱子、路障、机器人 | 运算、逻辑 |
| 方向地图 | 按指令移动到目标点 | 东西南北、坐标 |
| 反弹炮弹 | 利用墙面反弹击中目标 | 角度、反射 |
| Boss 训练 | 阶段式弱点挑战 | 综合能力 |

### 3.2 学习任务嵌入方式

```text
数学题 -> 获得炮弹、打开弹药箱、解除护盾
英语题 -> 找到正确补给、识别目标物
方向题 -> 找到隐藏路线、进入正确区域
逻辑题 -> 按顺序激活机关、选择正确开关
物理题 -> 使用反弹、角度、距离命中目标
```

### 3.3 最小可玩闭环

第一版 MVP 只需要实现：

- 1 个训练基地场景
- 1 辆默认坦克
- 3 个关卡
- 数学题驱动弹药箱或开门
- 简单敌方训练机器人
- 通关星级奖励
- 1 个可解锁涂装或坦克

## 4. 坦克系统

### 4.1 坦克风格

坦克可以参考经典坦克轮廓，但避免直接复刻真实型号和完整军事叙事。命名使用儿童友好的原创名称。

| 原型风格 | 游戏名称示例 | 特征 |
| --- | --- | --- |
| T-34 风格 | 雪原中坦 | 倾斜装甲、机动均衡 |
| 虎式风格 | 山岳重坦 | 方正车体、厚装甲、大炮管 |
| 谢尔曼风格 | 星盾中坦 | 高车身、圆炮塔、综合均衡 |
| 轻坦风格 | 风速侦察车 | 快速、高视野、低装甲 |
| 火炮风格 | 雷鸣支援车 | 远射程、低装甲、弹道特殊 |
| 工程车风格 | 铁臂工程车 | 推箱、修桥、清障 |

### 4.2 基础属性

每辆坦克有 5 个核心属性，每个属性 1-5 级。属性必须能直接反映在对局手感中。

| 属性 | 对局表现 |
| --- | --- |
| 火力 | 伤害、炮弹速度、装填速度、爆炸范围 |
| 机动 | 移动速度、转向速度、加速、倒车速度 |
| 装甲 | 生命值、正面减伤、碰撞减伤 |
| 隐蔽 | 敌人发现距离、草丛隐藏、首发加成 |
| 视野 | 发现敌人范围、小地图提示、提前看到目标 |

### 4.3 属性示例

```json
{
  "id": "mountain-heavy",
  "nameKey": "tank.mountainHeavy.name",
  "role": "heavy",
  "stats": {
    "firepower": 5,
    "mobility": 2,
    "armor": 5,
    "stealth": 1,
    "vision": 2
  },
  "defaultSkin": "forest-training"
}
```

### 4.4 升级材料

| 材料 | 提升 |
| --- | --- |
| 炮管零件 | 火力 |
| 发动机零件 | 机动 |
| 装甲板 | 装甲 |
| 迷彩网 | 隐蔽 |
| 观察镜 | 视野 |

## 6. 学习系统

### 6.0 模式分层

系统必须区分不同用户模式，题库、文案、奖励和 AI 解释策略按模式隔离。

| 模式 | 用户 | 内容边界 |
| --- | --- | --- |
| 儿童学习模式 | 6-12 岁 | 基础学科、正向反馈、无成人测试内容 |
| 青少年挑战模式 | 12-18 岁 | 更高难度知识题、逻辑题、空间推理 |
| 成人休闲模式 | 成人 | 百科挑战、策略题、趣味心理测试、亲子陪玩 |
| 家长管理模式 | 家长 | 查看报告、配置题库、设置游玩时长 |

儿童账号默认只能进入儿童学习模式。青少年和成人内容必须通过年龄档案或家长设置启用。

### 6.1 学科模块

| 模块 | 示例 |
| --- | --- |
| 数学 | 加减乘除、比较大小、应用题 |
| 英语 | 单词识别、颜色、动物、交通工具 |
| 方向 | 东西南北、坐标、路径 |
| 逻辑 | 顺序、分类、条件判断 |
| 物理 | 角度、反射、速度、距离 |
| 百科 | 坦克常识、地理、历史常识、科学常识 |
| 心理测试 | 性格倾向、决策风格、亲子互动测试，仅限成人/家长娱乐模式 |

心理测试必须标注为娱乐内容，不得输出医学诊断、心理疾病判断、治疗建议或高风险结论。

### 6.2 题目结构

题目必须来自 API、数据库、正式 seed 或配置文件，不允许写死在页面组件中。

```json
{
  "id": "math-001",
  "type": "math",
  "difficulty": 1,
  "promptKey": "question.math.addition",
  "promptParams": { "a": 8, "b": 7 },
  "answers": [
    { "id": "a", "text": "12" },
    { "id": "b", "text": "15" },
    { "id": "c", "text": "17" }
  ],
  "correctAnswerId": "b",
  "explanationKey": "explanation.math.addition"
}
```

### 6.3 难度调整

难度根据孩子表现自动调整：

- 连续答对：提高题目难度或减少提示
- 连续答错：降低难度、增加图形提示
- 某类型错误频繁：增加同类练习
- 过关但答题慢：保持难度，减少压力

## 7. AI 架构

### 7.1 AI 定位

AI 不直接控制游戏实时逻辑。AI 作为学习教练和内容辅助层，负责：

- 生成题目草稿
- 错题解释
- 个性化练习推荐
- 学习报告总结
- 家长任务生成
- 关卡学习目标建议

### 7.2 LangChain 使用边界

推荐使用 LangChain 组织 AI 工作流，但不要把 LangChain 放进前端游戏循环。

```text
前端游戏 -> 后端 API -> AI 服务层 -> LLM
                  │
                  ├── 题库服务
                  ├── 学习记录服务
                  ├── 难度策略服务
                  └── 安全过滤服务
```

### 7.3 AI 服务模块

| 模块 | 作用 |
| --- | --- |
| QuestionGeneratorChain | 根据年龄、学科、难度生成题目 |
| ExplanationChain | 用儿童能懂的话解释错题 |
| DifficultyPlannerChain | 根据历史表现推荐难度 |
| ParentReportChain | 生成家长学习报告 |
| SafetyGuard | 限制 AI 只输出儿童学习相关内容 |

### 7.4 AI 输出要求

AI 输出必须结构化，并经过后端校验后才能入库或返回前端。

```json
{
  "subject": "math",
  "difficulty": 2,
  "question": "9 + 6 = ?",
  "answers": ["13", "15", "16"],
  "correctAnswer": "15",
  "explanation": "9 加 6 可以先算 9 加 1 等于 10，再加 5 等于 15。"
}
```

后端必须校验：

- 答案唯一
- 正确答案可验证
- 难度符合范围
- 文案适合儿童
- 不包含不安全内容

## 8. 技术栈

### 8.1 第一阶段推荐

| 层 | 技术 |
| --- | --- |
| 前端 | TypeScript + React 或 Vue |
| 游戏引擎 | Phaser 4.2.1 |
| UI | Web Component / React / Vue 组件层 |
| 后端 | Node.js + NestJS 或 Java Spring Boot |
| 数据库 | PostgreSQL |
| 缓存 | Redis，可后置 |
| AI 编排 | LangChain |
| 鉴权 | JWT + 家长账号 |
| 国际化 | i18next / vue-i18n |
| 测试 | Vitest/Jest + Playwright |

如果后续明确要做 3D：

| 阶段 | 推荐 |
| --- | --- |
| 2D MVP | Phaser 4.2.1 |
| 2.5D | Phaser + 等距地图，或 Three.js 固定俯视镜头 |
| low-poly 3D | Three.js / React Three Fiber |
| 更复杂 3D | Unity |

### 8.2 视觉演进路线

第一版不要直接上完整 3D。视觉演进服从第 15 节产品阶段，不单独占用 Phase 2/3 编号：

```text
Phase 1-3: Phaser 2D 俯视角持续交付产品闭环
Phase 4: 2.5D 等距视角或 Three.js low-poly 3D 训练场
```

## 9. 前后端分离规范

### 9.1 前端职责

- 渲染游戏画面
- 播放动画和音效
- 接收玩家输入
- 调用 API 获取关卡、坦克、题目、奖励
- 展示学习反馈和升级结果

前端禁止：

- 在组件中硬编码正式题库
- 在组件中硬编码坦克成长规则
- 在组件中硬编码奖励掉落规则
- 绕过 API 直接决定学习进度

### 9.2 后端职责

- 用户、孩子档案、家长档案
- 题库、关卡、坦克、涂装配置
- 学习记录、答题记录
- 奖励结算
- 坦克升级
- AI 内容生成与审核
- 多语言内容管理

### 9.3 API 示例

```text
GET    /api/children/:childId/profile
GET    /api/tanks
GET    /api/tanks/:tankId
GET    /api/levels/:levelId
POST   /api/levels/:levelId/start
POST   /api/sessions/:sessionId/answer
POST   /api/sessions/:sessionId/finish
POST   /api/tanks/:tankId/upgrade
GET    /api/children/:childId/report
POST   /api/ai/questions/generate
POST   /api/ai/explanations
```

## 11. 多主题与多语言

### 11.1 多主题

主题包括视觉和关卡氛围，但不能影响核心学习规则。

建议主题：

- 训练基地
- 森林营地
- 沙漠靶场
- 雪地补给线
- 太空训练场

主题配置示例：

```json
{
  "id": "snow-field",
  "nameKey": "theme.snowField.name",
  "palette": {
    "primary": "#4f7fbf",
    "ground": "#dceaf5",
    "accent": "#f7c948"
  },
  "tileSet": "snow",
  "music": "snow-training-loop"
}
```

### 11.2 多语言

所有可见文字必须使用 i18n key：

```text
tank.mountainHeavy.name
level.forestSupply.title
question.math.addition
reward.part.cannon
```

不要在组件里直接写中文或英文业务文案。允许按钮类通用文案使用 i18n：

```text
action.start
action.pause
action.continue
action.upgrade
```

### 12.2 文件大小约束

- 单个组件建议不超过 250 行

- 样式按组件或模块拆分
- 配置数据放 config 或由 API 返回

## 13. 代码风格

### 13.1 通用要求

- TypeScript 优先
- 明确类型，不滥用 `any`
- API response 定义 DTO 类型
- 业务规则写在 service/system，不写在 UI 组件
- 命名使用完整语义，不用模糊缩写
- 函数保持单一职责
- 不做超大文件

### 13.2 AI 开发约束

后续让 AI 改代码时必须遵守：

1. 先查已有模块、API、配置、数据模型，不重复实现已有能力。
2. UI 可以参考原型和视觉风格，但数据必须来自 API、配置或数据库。
3. 不允许为了演示效果在页面组件中写死坦克、题目、奖励、用户数据。
4. 新增功能必须说明数据来源。
5. 涉及关卡、题库、奖励、坦克参数时，必须同步更新类型、API、测试。
6. 大功能拆成小 PR，不把 UI、数据库、AI、部署混在一个 PR。

- 星级奖励

- i18n 和主题基础框架

### Phase 2: 养成与学习记录

- 多坦克
- 零件升级
- 涂装解锁
- 答题记录
- 家长报告基础版
- 英语和方向任务

### Phase 3: AI 学习教练

- LangChain 接入
- AI 生成题目草稿
- 错题解释
- 自适应难度
- 家长报告总结
- 内容安全审核

### Phase 4: 2.5D / low-poly 3D

- 坦克 2.5D 或 Three.js 展示
- 更高识别度的经典风格坦克
- 涂装系统升级
- 场景主题增强

## 16. 验收指标

### 16.1 游戏性指标

- 6-10 岁儿童无需复杂说明即可开始移动和射击
- 每关 3-5 分钟内完成
- 属性升级后手感可感知
- 坦克外观能被孩子区分

### 16.2 学习指标

- 每局至少完成 3 个学习任务
- 错题有解释
- 家长能看到学习记录
- 难度能随表现变化

### 16.3 工程指标

- 前端无硬编码正式题库
- 关卡、题目、坦克参数、奖励规则可配置或来自 API
- 支持多语言和多主题
- 单文件不过大
- 核心逻辑有测试
- 构建、lint、类型检查通过

## 17. 开发时的默认优先级

当实现出现冲突时，按以下顺序取舍：

1. 儿童安全
2. 游戏闭环完整
3. 学习任务自然嵌入
4. 数据配置化和前后端分离
5. 坦克外观识别度
6. AI 个性化
7. 3D 表现

第一版宁可少做坦克和关卡，也要保证“开坦克、完成学习任务、获得奖励、升级坦克”这个闭环真实跑通。
