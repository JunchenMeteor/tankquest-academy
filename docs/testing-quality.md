# 测试与质量规范

## 1. 测试目标

保证游戏闭环、后端规则、AI 输出和资源加载可验证。

## 2. 测试类型

```text
unit: 纯函数、数值、奖励、难度
contract: API 请求响应
integration: 后端模块和数据库
e2e: 浏览器真实流程
game: 移动、射击、碰撞、敌人状态机
ai: 输出 schema 和安全规则
asset: 模型、贴图、地图加载
```

## 3. 必测逻辑

- 答题判定。
- 奖励结算。
- 坦克升级。
- 难度调整。
- 家长控制。
- 儿童/成人模式隔离。
- 对局重复提交。
- AI 输出校验。
- 资源 hash 校验。

## 4. E2E 场景

```text
登录
选择孩子档案
选择坦克
开始关卡
完成一道题
击败敌人
结束关卡
领取奖励
升级坦克
查看进度
```

## 5. 质量门禁

PR 合入前至少通过：

```text
lint
typecheck
unit tests
API contract tests
build
critical e2e smoke
```

## 6. AI 开发要求

AI 修改代码后必须说明：

- 改了哪些模块。
- 数据来源是什么。
- 跑了哪些测试。
- 哪些测试没跑。
- 是否涉及儿童安全、奖励、AI 输出或隐私。
