# Phase 3 AI 学习教练范围

> 状态：2026-07-16 已通过工程验收，目标版本 v0.3.0。Phase 3 只交付 AI 学习辅助；2.5D/3D、客户端打包和多人玩法属于后续阶段。

## 1. 目标

Phase 3 在 v0.2.0 的后端权威学习闭环上增加可降级的 AI 辅助：

```text
正式题库与后端判题
-> AI 提供结构化解释、难度建议和家长摘要
-> 主后端校验、裁决或回退到确定性模板
-> 游戏、奖励和学习记录在 AI 故障时继续工作
```

AI 是学习教练和内容草稿工具，不是游戏裁判、自由聊天机器人或业务数据库写入者。

## 2. 必须实现

### 2.1 独立 AI 服务

- 使用 FastAPI 和 Pydantic 提供结构化服务契约。
- LangChain 只位于 AI 服务内部，不进入 Web、Phaser 或主后端实时循环。
- 外部模型通过供应商适配层接入；默认允许安全模板供应商，任何供应商失败都返回可识别的降级结果。
- 提供健康检查、严格超时、请求标识和最小可观测日志。
- 不在腾讯云主机本地运行大模型，不把密钥写入镜像、仓库或响应。

### 2.2 AI Gateway

- NestJS 主后端是 Web 和 AI 服务之间的唯一网关。
- 网关只发送完成任务所需的最小字段，不发送姓名、邮箱、出生年份、地址、学校、原始事件日志或坦克战斗日志。
- AI 输出必须经过共享 schema、长度、枚举和安全规则校验后才能返回或成为建议。
- 超时、网络错误、无效 JSON 和安全拒绝统一降级，不阻断答题、结算、报告或下一局。

### 2.3 错题解释

- 后端完成正确性判定后，AI 才能根据题目、正确答案、孩子选择、技能、难度和语言生成解释。
- 解释简短、非责备、适龄，只给一步提示或拆分思路。
- AI 不能改变 `correct`、正确答案、弹药反馈或奖励。
- AI 不可用时返回题库中的正式解释或确定性模板。

### 2.4 自适应难度与练习建议

- 主后端根据学习记录确定允许的难度范围和最终选择。
- AI 只能在当前正确率、平均耗时、尝试数和家长控制范围内给出结构化建议。
- 主后端必须钳制越界建议，并记录采用、调整或拒绝原因。
- 缺少足够学习数据或 AI 不可用时保持确定性难度策略。

### 2.5 家长报告摘要

- AI 只读取现有家长报告的聚合学科、技能、正确率、耗时和完成局数。
- 摘要包含练习内容、明显进步、需要关注和下一步建议。
- 不包含心理诊断、能力标签、医学判断、焦虑式文案或儿童身份字段。
- AI 摘要失败时保留 v0.2.0 的确定性家长报告和建议。

### 2.6 题目草稿与安全审核

- AI 服务支持生成结构化题目草稿，但草稿不能直接成为 `published` 题目。
- 草稿必须包含学科、技能、难度、题干、候选答案、唯一正确答案和解释。
- 主后端校验 schema、唯一答案、范围、长度和儿童安全；无法确定正确性的内容必须拒绝。
- 在正式管理员鉴权和审核 UI 完成前，不向公网提供“自动发布题目”接口。

## 3. 数据与安全边界

允许发送给 AI：

```text
ageGroup（枚举）
locale（枚举）
subject / skillKey / difficulty
题干与候选答案
单次选择和后端已判定的正确答案
聚合尝试数、正确率、平均耗时、完成局数
```

禁止发送给 AI：

```text
姓名、昵称、邮箱、出生年份
家庭、学校、地址和设备标识
自由聊天历史
原始对局事件、坐标和战斗日志
数据库主键或供应商密钥
```

AI 服务不得直接连接业务 PostgreSQL，不得发放奖励、修改升级、写学习记录或发布内容。

## 4. 明确不做

- 自由聊天、情感陪伴、心理测试或医疗建议。
- AI 判题、AI 奖励、AI 通关和 AI 实时控制敌人。
- 自动发布 AI 题目或绕过人工审核。
- 正式多家庭登录、管理员后台全量功能。
- 本地部署大模型、向量数据库、复杂 Agent 或多 Agent 工作流。
- Phase 4 的 2.5D/3D、资源美术升级和客户端打包。

## 5. 验收标准

- AI 服务健康检查、结构化契约和安全拒绝测试通过。
- 主后端只通过 AI Gateway 调用服务，并对超时、无效输出和拒绝执行确定性降级。
- 错题解释不改变后端判题和奖励，AI 关闭时完整学习闭环仍通过。
- 自适应难度最终值始终处于家长控制和后端策略允许范围。
- 家长摘要只使用聚合数据，儿童游戏界面不显示家长摘要。
- 题目草稿不能直接进入正式题库或游戏会话。
- preview/release 在没有外部模型密钥时仍健康；配置供应商后不向客户端暴露密钥或原始响应。
- 单元、契约、降级、安全、数据库、浏览器和 Docker 部署检查通过。

## 6. PR 顺序

1. Phase 3 范围、边界和验收标准。
2. FastAPI AI 服务、结构化契约、题目草稿能力、供应商适配与 SafetyGuard。
3. NestJS AI Gateway、超时降级、最小数据映射和 Docker 部署。
4. 错题解释接入、语言适配和后端判题隔离测试。
5. 后端裁决的自适应难度与练习建议。
6. 家长报告 AI 摘要和聚合数据隐私测试。
7. Phase 3 验收证据、版本发布和生产验证。

每个 PR 必须从最新 `main` 创建，等待 `Verify` 和容器检查通过后 squash merge；不得直接 push `main` 或 `release`。
