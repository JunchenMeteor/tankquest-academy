# Phase 1.6 命中判定收口

> 状态：已于 2026-07-15 完成工程验收，发布版本为 v0.1.2。

## 1. 目标

Phase 1.6 只解决炮弹命中结果缺少层次的问题，让火力、装甲、命中方向和入射角对战斗产生可解释影响。

```text
炮弹命中 -> 计算命中区域和入射角 -> 比较穿深与有效装甲
-> 跳弹 / 未击穿 / 击穿 -> 显示反馈并记录战斗日志
```

玩家和敌方坦克使用同一套确定性规则。撞击伤害保持独立，不参与穿深计算。

## 2. 必须实现

- 火力分别影响炮弹穿深和击穿后的基础伤害。
- 装甲属性产生正面、侧面、背面三组基础装甲值。
- 命中区域由炮弹方向和坦克朝向计算，不使用随机数。
- 过斜入射角产生跳弹，穿深不足产生未击穿，穿深足够产生击穿。
- 跳弹和未击穿不扣生命值；击穿伤害仍受装甲缓冲，但至少造成可感知伤害。
- 玩家与敌人都显示清晰的中文命中结果、伤害数字和短暂视觉反馈。
- 结构化战斗日志记录结果、区域、穿深、有效装甲、伤害和剩余生命值，不记录儿童数据。
- 单元测试覆盖边界值，Playwright 覆盖连续受击后游戏循环仍运行。

## 3. 明确不做

- 随机暴击。
- 弹药架、发动机、履带等部件损坏。
- 炮弹类型和弹药库存。
- 真实军用穿甲公式。
- 付费弹药、随机强化或成人化暴力表现。

## 4. 验收标准

- 同一输入永远得到同一命中结果。
- 正面装甲比侧面和背面更难击穿。
- 升级火力提高穿深和伤害，升级装甲提高有效装甲。
- 至少能稳定复现一次跳弹、一次未击穿和一次击穿。
- 敌我双方规则一致，生命值不会出现 `NaN`、负数或命中后停帧。
- 格式、schema、lint、typecheck、unit、build 和关键 E2E 全部通过。

## 5. PR 顺序

1. 纯函数穿深和方向装甲模型。
2. Phaser 命中集成、反馈和日志。
3. Phase 1.6 验收证据和发布。

## 6. 验收证据

- 纯函数测试覆盖正面、侧面、背面、跳弹、未击穿、击穿和异常输入。
- 玩家和敌方炮弹都携带由坦克火力推导的穿深，并使用同一命中函数。
- 轻型敌人正面命中中型玩家时稳定产生未击穿，生命值保持 `150/150`。
- 高火力敌人正面连续击穿时生命值稳定下降，游戏画布和输入循环继续运行。
- 结构化日志包含命中结果、区域、角度、穿深、有效装甲、伤害和剩余生命值。
- `TrainingScene` 已拆回 400 行以内，炮弹和撞击处理位于独立 system。
- 格式、schema、lint、typecheck、unit、build、Playwright 和 Docker 镜像验证通过。
