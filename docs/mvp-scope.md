# MVP 范围与验收标准

> 状态：第一阶段工程 MVP、Phase 1.5 战斗反馈和 Phase 1.6 命中判定已完成，家庭试用指标待真实儿童和家长验证。逐项证据见 `phase-one-mvp-acceptance.md`。

## 1. MVP 目标

第一版目标不是做完整 3D 坦克游戏，而是验证核心闭环：

```text
开坦克 -> 完成学习任务 -> 获得奖励 -> 升级/解锁 -> 继续挑战
```

## 2. 必须实现

- Web 端可运行。
- 1 个训练基地主题。
- 1-3 辆坦克。
- 3 个关卡。
- 数学题任务。
- 简单敌方训练机器人。
- 坦克移动、炮塔旋转、射击、碰撞。
- 对局开始和结束。
- 星级评价。
- 零件奖励。
- 坦克属性升级。
- 家长/孩子基础档案。
- 后端结算奖励。
- 题库和关卡来自 API 或正式配置。

## 3. 明确不做

- 实时多人联机。
- 完整 3D 写实坦克。
- 本地部署大模型。
- 复杂商城。
- 抽卡。
- 大规模题库后台。
- 移动端发布。
- Windows 打包。
- 心理测试。

## 4. MVP 验收

- 孩子能无说明完成移动和射击。
- 一局 3-5 分钟。
- 至少 3 道题嵌入关卡目标。
- 通关后能看到奖励。
- 奖励能升级坦克属性。
- 升级后手感有变化。
- 刷新页面后进度不丢。
- 构建、测试、接口检查通过。

## 5. MVP 成功指标

- 孩子愿意重复玩 3 局以上。
- 家长能理解孩子学了什么。
- 至少一个属性升级能被明显感知。
- 题目不是突兀弹窗，而是关卡资源的一部分。
