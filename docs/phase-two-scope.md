# Phase 2 养成与学习记录范围

## 1. 目标

Phase 2 在已完成的单局闭环上增加可持续养成和家长可理解的学习结果：

```text
选择已拥有坦克和涂装 -> 完成不同学科任务 -> 后端记录学习表现
-> 获得与解锁 -> 家长查看基础报告 -> 继续选择合适任务
```

## 2. 必须实现

### 2.1 多坦克

- 后端返回孩子已拥有的坦克，不允许选择未拥有或未启用坦克。
- 至少提供轻型、中型、重型三辆原创训练坦克。
- 坦克在火力、机动、装甲、隐蔽和视野上有可感知差异。
- 对局、升级和结算使用用户实际选择的坦克。

### 2.2 涂装

- 涂装、解锁条件、孩子所有权和当前装备由后端持久化。
- 至少提供每辆坦克的默认涂装和一个可解锁原创涂装。
- Phase 2 涂装只改变外观，不提供战斗数值加成。
- 不使用真实军徽、真实编号或受版权保护素材。

### 2.3 学习记录

- 后端按孩子、学科和技能聚合尝试数、正确数、平均答题时长和当前难度。
- 答题提交和对局结算保持幂等，不重复累计。
- 提供孩子学习进度查询 API，前端不自行推导权威记录。

### 2.4 家长报告基础版

- 报告展示时间范围、完成对局数、学科正确率、平均答题时长和最近技能表现。
- 报告只读取家庭内当前孩子数据，并与儿童游戏界面分离。
- 第一版报告使用确定性模板，不接入 AI 总结。
- 不展示真实姓名、学校、地址或原始敏感日志。

### 2.5 英语和方向任务

- 题目内容来自后端题库和正式配置。
- 英语至少覆盖基础词义匹配，方向至少覆盖左右和方位判断。
- 三类学科共用答案校验、记录、反馈和结算边界。
- 错误反馈不羞辱儿童，不把答案判定交给 AI。

## 3. 明确不做

- AI 学习教练和生成题目。
- 2.5D/3D 画面升级。
- 商城、抽卡、付费数值和限时刺激。
- 正式多家庭鉴权系统和管理后台全量功能。
- 移动端或 Windows 打包。

## 4. 验收标准

- 孩子能在三辆已拥有坦克中选择并启动对局。
- 涂装解锁和装备刷新后不丢失，且不改变战斗数值。
- 数学、英语和方向任务均由后端判定并进入学习记录。
- 家长报告能解释孩子完成了什么、正确率如何和下一步关注的技能。
- 未拥有坦克或涂装请求被后端拒绝。
- 儿童游戏界面不能直接进入家长报告。
- 数据库迁移、契约、幂等、安全边界和浏览器闭环测试通过。

## 5. PR 顺序

1. 多坦克所有权、选择和运行时差异。
2. 涂装数据、解锁、装备和渲染。
3. 学习记录聚合与进度 API。
4. 家长报告 API 和独立 UI。
5. 英语与方向题库、关卡和端到端流程。
6. Phase 2 验收证据、版本发布和生产验证。
