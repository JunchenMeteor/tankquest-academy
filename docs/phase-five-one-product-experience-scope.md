# Phase 5.1 内容与产品体验升级范围

> 状态：待实施。目标版本 v0.6.0。Phase 5.1 不增加新平台，而是把已通过 Phase 1-5 工程验收的能力收敛成内容足够、反馈清晰、愿意重复游玩的产品体验。

## 1. 背景与验收口径

Phase 1-5 已经建立后端权威规则、战斗闭环、学习记录、安全 AI、2.5D Web/PWA 和 Windows 技术预览，但“工程门禁通过”不等于“儿童实际体验成熟”。当前正式内容仍然接近纵向切片：关卡和题目数量有限，部分地图复用同一训练主题，AI 能力主要位于后台，Windows 壳也不会改变 Web 游戏内容。

从 Phase 5.1 开始同时维护两类结论：

```text
工程验收：契约、规则、测试、构建、部署和安全边界通过
产品体验验收：内容不单调、战斗行为合理、升级可感知、儿童愿意重复游玩
```

只有两类验收都通过，才能把当前纵向切片称为完成产品体验收口。

## 2. 目标体验

```text
选择差异明确的坦克和任务
-> 在目标、地形和敌人组合不同的关卡中战斗
-> 完成与场景动作相关的学习任务
-> 获得清晰反馈和可感知升级
-> 由下一难度和新内容驱动重复游玩
```

Phase 5.1 优先提升内容密度、玩法变化和反馈质量，不通过完整 3D、多人联机或更多客户端掩盖当前产品深度不足。

## 3. 必须实现

### 3.1 分层学习内容

- 数学、英语和方向各自提供至少 4 个可辨识难度层级，并覆盖不止一个技能点。
- 正式题库至少包含 120 个经过确定性答案校验的本地化题目或题目变体；英中题干、选项和解释不能只翻译 UI 外壳。
- 同一局不得重复题目；内容池足够时，连续三局不得重复整组题目。
- 自适应难度必须选择真实存在的已发布内容，并在内容不足时明确保持当前难度，不能伪造个性化结果。
- AI 可以生成待审核草稿和解释，但不得自动发布儿童可见题目；正式内容仍需确定性校验和审核状态。

### 3.2 关卡与目标变化

- 至少提供 12 个已发布关卡，三类学科都具有从入门到挑战的任务路径。
- 至少实现 5 类可观察目标组合，例如歼灭、波次防守、护送/补给、路线选择和精英目标。
- 地图不能只更换障碍坐标；主题、通路、掩体、出生点、敌人波次和学习交互需要共同形成差异。
- 训练基地、森林营地和雪地补给线都必须有默认关联关卡，用户不手动切主题也能看到场景变化。
- 学习任务优先与门、路线、补给和目标选择结合，减少脱离战斗语境的普通答题弹层感。

### 3.3 敌人感知与受袭反应

- 被动发现范围继续受敌人视野、玩家隐蔽、关卡难度和遮挡规则约束。
- 玩家首次从隐蔽或远距离开火可以获得先手优势，但不能在正常炮弹射程内持续攻击完全无反应的敌人。
- 炮弹命中、近距离擦过或附近友军受击后，敌人进入有限时长的警戒状态，面向威胁来源并追踪最后已知位置。
- 警戒可按合理半径通知附近同组敌人，但不能让全地图敌人瞬间获得永久精确坐标。
- 敌人脱离接触后按明确的记忆时间、搜索范围和撤回条件恢复巡逻，避免永久追击和边界卡死。
- 为被动发现、先手攻击、受击警戒、友军告警、记忆失效和重新发现增加确定性测试。

### 3.4 战斗与成长反馈

- 火力、机动、装甲、重量、视野和隐蔽的变化必须映射到可测量的战斗结果，并在选择页和结算页解释差异。
- 升级前后至少有一项即时对比，例如装填时间、炮弹伤害、速度、转向、生命、穿深或撞击表现。
- scout、medium 和 heavy 敌人在移动、发现、开火、装甲和受击反馈上具有可观察差异，不只更换轮廓和颜色。
- 增加明确的开火、装填、命中、跳弹、未击穿、击穿、低血量和摧毁反馈；重要状态不能只依赖文字日志。
- 结算必须说明学习表现、战斗表现、奖励来源、升级效果和下一推荐任务。

### 3.5 视觉、声音与页面体验

- 统一首页、坦克选择、任务选择、战斗 HUD、答题反馈、结算和升级页面的视觉层级。
- 使用版本化、可审核的原创坦克、地形、图标、特效和音效资源；资源失败时继续保留安全降级。
- 三种主题在颜色以外还要通过地面、障碍、装饰、环境效果和声音形成差异。
- 儿童首次进入时可以在不阅读长说明的情况下理解移动、瞄准、开火、答题和任务目标。
- 桌面和触控布局都必须保持主要状态可读，不以缩小桌面 UI 代替移动适配。

### 3.6 产品体验验证

- 自动化继续覆盖答案、奖励、升级、碰撞、穿深、敌人炮弹、PWA 和平台边界。
- 增加内容重复率、关卡目标、敌人警戒、属性升级差异和关键反馈的可执行验收。
- 在标记产品体验通过前，至少完成 3 组儿童与家长监督下的家庭试用，记录首次操作障碍、单局时长、重复游玩、升级感知和家长理解度。
- 家庭试用不得记录不必要的儿童身份信息；问题以匿名观察结论进入 Issue。

## 4. 明确不做

- 完整 low-poly 3D、自由镜头、3D 物理或更换游戏引擎。
- 多人匹配、实时同步、排行榜、商城、抽卡或付费数值。
- Android/iOS 打包、Windows 商店、代码签名或自动更新。
- AI 自动发布题目、AI 判题、AI 发奖励或 AI 实时控制敌人。
- 正式多家庭鉴权和大规模运营后台；相关能力继续独立评估。

## 5. 验收标准

- 新用户无需长说明即可开始第一局，并能理解当前目标和战斗/学习反馈。
- 三类学科、四档难度、至少 12 个关卡和 120 个正式本地化题目/变体可通过后端查询并完成。
- 连续三局在内容池充足时不会出现完全相同的题目组合。
- 玩家可以利用隐蔽获得先手，但无法通过站在被动发现范围外持续无伤攻击不反应的敌人。
- 三辆玩家坦克、三类敌人和关键属性升级在实测数据与主观操作上都可区分。
- 三种主题和五类目标组合在默认任务路径中可遇到，不依赖隐藏设置或开发入口。
- 完整学习、战斗、结算、升级、重玩和家长报告流程在 Web/PWA 与 Windows 技术预览中保持同一权威结果。
- 自动化门禁、preview/release 部署探针和家庭试用门槛全部通过后，路线图才能标记“产品体验已验收”。

## 6. 建议 PR 顺序

1. Phase 5.1 范围、双重验收口径和路线图状态。
2. 敌人受袭警戒、感知记忆和远距离无反应攻击修复。
3. 本地化题目模型、内容校验和扩展题库。
4. 多目标关卡运行时、关卡路径和默认主题分配。
5. 坦克/敌人差异、升级对比和结算说明。
6. 页面视觉系统、战斗特效、动画和音效资源。
7. 内容重复率、关键旅程、触控和性能回归门禁。
8. 家庭试用问题收口、Phase 5.1 验收记录和 v0.6.0 发布。

每个 PR 继续从最新 `main` 创建，等待受保护 `Verify` 和相关容器检查通过后 squash merge；不得直接 push `main` 或 `release`。
